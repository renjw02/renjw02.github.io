谓词变换

谓词变换（Predicate Transformer）：给定一条断言和一条程序语句，通过变换得到另一条断言。

常见的谓词变换有：

  • 最弱前置条件（Weakest Precondition）wp(st, ψ)：假设语句执行后满足 ψ，执行语句前必须满足的条件。

    1. {wp(st, ψ)} st {ψ} 是有效式，即 wp(st, ψ) 是一个能够让 ψ 成
      立的前置条件，且

    2. 若存在 φ 使得 {φ} st {ψ} 是有效式，则 φ ⇒ wp(st, ψ)，即
      wp(st, ψ) 是所有此类前置条件中最弱的。

  • 最强后置条件 （Strongest Postcondition）sp(φ, st)：假设语句执行前满足 φ，执行语句后必定满足的条件。

    1. {φ} st {sp(φ, st)} 是有效式，且

    2. 若存在 ψ 使得 {φ} st {ψ} 是有效式，则 sp(φ, st) ⇒ ψ，即
      sp(φ, st) 是所有此类后置条件中最强的。

验证条件

给定程序 st 和规约 (φ, ψ)，验证条件 (verification conditions) 是一组一阶逻辑公式，满足：

如果所有验证条件都是有效式，则程序满足规约

最弱前置条件提供了一种生成验证条件的方法：

  1. 从程序规约的后置条件 ψ 开始

  2. 由后向前计算程序的最弱前置条件 wp(st, ψ)

  3. 验证 φ → wp(st, ψ) 是否为有效式

最弱前置条件计算

计算赋值语句的wp时可能会遇到的问题

当公式 φ 中存在与 x 同名的约束变元时，如何避免 φ[x → e] 发生变元捕获？
α-替换：在替换之前先重命名 φ 中可能导致捕获的约束变元
更一般的替换形式是：φ[G0 → F0, · · · , Gn → Fn], 其中 Fi, Gi 也都是公式。为避免变元捕获，令

对 φ 中的每一个约束变元 x，若 $x ∈ V_{free}$，则以一个新的变元符号来重命名 x；
以 φ′ 表示重命名后的公式，对 φ′ 同时应用所有替换；
如果 Gi 和 Gj 在 φ′ 中的某处重合，并且 Gi 是 Gj 的子公式，则只应用范围较大的替换，即 Gj → Fj。

验证条件

基于最弱前置条件的程序验证

  1. 计算 φ′ = wp(st, ψ)

  2. 计算 VC(st, ψ)

  3. 检查 φ → φ ′ 是否有效式

  4. 检查 VC(st, ψ) 中的每一个公式是否有效式

  • VC(st, ψ) 和 φ → φ′ 统称为验证条件（Verification Conditions)

  • 如果所有验证条件的检查都通过，则 st |= (φ, ψ)

  • 若 st |= (φ, ψ)，不一定能够通过所有验证条件的检查
